# View — шаблонизатор cwfr

Модуль рендерит текстовые шаблоны (HTML и любые другие) по данным из
`json_doc_t`. Синтаксический разбор выражений выполняется методом
рекурсивного спуска (`viewexpr.c`), значения типизированы: строки, числа,
булевы, null и JSON-контейнеры. Распарсенные шаблоны кэшируются в
`viewstore` — файл читается и разбирается один раз, дальше рендер идёт по
готовому дереву тегов.

## Быстрый старт

### Из обработчика HTTP

```c
#include "json.h"

void get(httpctx_t* ctx) {
    json_doc_t* doc = json_create_empty();
    json_token_t* root = json_token_alloc(JSON_OBJECT);
    json_set_root(doc, root);
    // ... наполнить документ данными ...

    // storage "views", файл <root>/index.html
    ctx->response->send_view(ctx->response, doc, "views", "/index.html");

    json_free(doc);
}
```

Путь принимает printf-формат: `send_view(..., "views", "/user_%d.html", id)`.
При ошибке разбора шаблона клиенту уйдёт строка `Render view error`.

### Напрямую

```c
#include "view.h"

char* html = render(doc, "views", "/index.html");
if (html != NULL) {
    // ... использовать ...
    free(html); // владение у вызывающего
}
```

`render` возвращает `NULL`, если шаблон не найден, синтаксически неверен или
не настроен `viewstore`. Отсутствующие в документе переменные ошибкой не
считаются — они рендерятся пустой строкой.

### Настройка storage

Шаблоны читаются из именованного storage (`config.json`, секция `storages`):

```json
"storages": {
    "views": {
        "type": "filesystem",
        "root": "/path/to/app/views"
    }
}
```

Подходит и `s3`-storage — модуль работает через общий интерфейс `storage`.

## Синтаксис шаблонов

Три вида тегов:

| Тег | Назначение |
|-----|------------|
| `{{ выражение }}` | вывод значения |
| `{% ... %}` | условия и циклы |
| `{* include /path *}` | вставка другого шаблона |

Одиночные `{`, `}`, `%`, `*` — обычный текст, экранировать не нужно.

### Вывод значений

```html
<h1>{{ user.name }}</h1>
<p>Цена: {{ price * (1 - discount) }}</p>
<p>Первый тег: {{ user.tags[0] }}, следующий: {{ items[pos + 1].title }}</p>
```

Правила вывода по типу результата:

| Тип | Вывод |
|-----|-------|
| строка | как есть |
| число | целые без дробной части (`42`), дробные — кратчайшая точная форма (`9.99`) |
| bool | `true` / `false` |
| null, отсутствующее значение | пустая строка |
| объект / массив | пустая строка (выводите поля или обходите циклом) |

### Условия

```html
{% if user.age >= 18 && user.role == 'admin' %}
    админ
{% elseif user.age >= 18 %}
    взрослый
{% else %}
    несовершеннолетний
{% endif %}
```

Условие — любое выражение. Истинность значения:

* **истинно**: непустая строка, ненулевое число, `true`, непустой объект/массив;
* **ложно**: `''`, `0`, `false`, `null`, пустой контейнер, отсутствующее значение.

### Циклы

```html
{% for item in items %}
    {{ index + 1 }}. {{ item.title }}
{% endfor %}

{% for value, key in settings %}
    {{ key }} = {{ value }}
{% endfor %}
```

* `{% for элемент in коллекция %}` — обход массива или объекта.
* Второе имя после запятой — ключ: для массива это числовой индекс,
  для объекта — строковый ключ.
* Без явного ключа доступна переменная `index` (число, участвует
  в арифметике).
* Коллекция — любое выражение: `{% for x in data.rows[2] %}`.
* Циклы вкладываются; во вложенном доступны переменные всех внешних циклов.
* Имена элемента и ключа перекрывают одноимённые поля корня документа.

### Вставка шаблонов

```html
{* include /partials/header.html *}
```

Путь — внутри того же storage. Вставка рекурсивная, глубина ограничена 100
(защита от самовключения).

## Выражения

### Литералы

```
42   9.99   -5.5e2   'строка'   "тоже строка"   true   false   null
```

В строках работают `\'`, `\"`, `\\`, `\n`, `\t`, `\r`.

### Переменные

Идентификатор (`[A-Za-z_][A-Za-z0-9_]*`), затем путь из `.поле` и
`[индекс]`. Индекс — любое выражение: числовое для массива, строковое для
объекта (`obj[key]`, `obj['literal']`). Выход за границы массива, обращение
к полю скаляра, отрицательный или дробный индекс дают null, а не ошибку.

### Операторы (по убыванию приоритета)

| Приоритет | Операторы | Замечания |
|-----------|-----------|-----------|
| 1 | `!` `-` (унарные) | `!` — логическое отрицание любого значения, `-` — только числа |
| 2 | `*` `/` `%` | только числа; `/ 0` и `% 0` дают null |
| 3 | `+` `-` | только числа (конкатенации строк нет) |
| 4 | `<` `<=` `>` `>=` | числа — численно, строки — лексикографически; разные типы не упорядочены (false) |
| 5 | `==` `!=` | значения разных типов не равны (`'1' != 1`) |
| 6 | `&&` | короткозамкнутый |
| 7 | `\|\|` | короткозамкнутый |

Скобки `( ... )` группируют как обычно. Одиночные `=`, `&`, `|` —
синтаксическая ошибка. Глубина вложенности выражения ограничена 32.

```html
{% if (a + b) * 2 > limit && !(banned || muted) %} ... {% endif %}
{% if count % 2 == 0 %}чётный{% endif %}
{{ matrix[i][j] * k }}
```

Операции над несовместимыми типами (`'a' + 1`) вычисляются в null — рендер
не падает, значение выводится пустым.

## Ошибки

* **Ошибка синтаксиса** (незакрытый тег, `{{ a + }}`, `{% unknown %}`,
  незакрытая строка) — весь `render` возвращает `NULL`, причина пишется в
  лог (`log_error`).
* **Ошибка данных** (нет переменной, индекс за границей, деление на ноль,
  несовместимые типы) — выражение даёт null: пустой вывод, ложное условие,
  пропущенный цикл.

## Низкоуровневый API

Обычно не нужен — `render` делает всё сам. Для инструментов и тестов:

```c
// разбор шаблона в дерево тегов
viewparser_t* parser = viewparser_init("views", "/index.html");
if (viewparser_run(parser)) {
    view_tag_t* root = viewparser_move_root_tag(parser); // забрать владение
    // ...
    root->free(root);
}
viewparser_free(parser);

// разбор и вычисление одного выражения
viewexpr_node_t* expr = viewexpr_parse("a.b[0] + 1", 10);
viewexpr_scope_t scope = { my_resolve, my_data }; // резолвер имён
viewexpr_value_t value = viewexpr_eval(expr, &scope);
// value заимствует строки у AST и документа — использовать до освобождения
viewexpr_node_free(expr);
```

`viewexpr_scope_t.resolve` получает первое имя пути (`a` в `a.b[0]`) и
возвращает `viewexpr_value_t`; остальную часть пути вычислитель проходит
сам по JSON-токенам. Рендер использует это для переменных циклов: сначала
ключ/элемент объемлющих `for`, затем поля корня документа.

## Кэширование

Первый `render` пути кладёт разобранное дерево в `viewstore` (потокобезопасно,
под блокировкой). Повторные рендеры того же пути файл не перечитывают.
Изменили шаблон на диске — перезапустите сервер.
